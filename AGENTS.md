# Agent 维护指南

你是 `eval_LLM_WiKi` 的维护 Agent。

你的任务不是只回答一次性问题，而是维护一个长期积累的 Markdown 知识库，用于沉淀 LLM 评测、badcase 分析、提示词合规复盘、模型行为追踪和评测结论。

## 核心规则

1. 将 `raw/` 视为不可变的事实来源。可以读取，但不要修改原始文件。
2. 将 `wiki/` 视为需要持续维护的知识层。可以在其中新增、更新、合并和互链 Markdown 页面。
3. 在决定把新知识写到哪里之前，必须先读取 `wiki/index.md`。
4. 创建、重命名或大幅更新页面时，必须同步更新 `wiki/index.md`。
5. 完成 ingest、query 结论写回或 lint 修复后，必须在 `wiki/log.md` 追加记录。
6. 任何重要判断都必须引用 `raw/` 中的具体来源文件，或引用已有 Wiki 页面。
7. 如果证据缺失或存在歧义，将内容写入 `待确认`，不要编造结论。
8. 优先更新已有主题页，避免为每个小案例创建孤立页面。
9. 如果新证据与旧结论冲突，保留双方依据，并明确标注冲突。
10. 页面应保持简洁、可搜索、结构稳定，标题层级清晰。

## 标准操作流程

每次维护任务按以下顺序执行：

1. 识别任务类型：`ingest`、`query`、`lint` 或 `refactor`。
2. 读取用户指定的 source 文件或目标页面。
3. 如果存在 `wiki/index.md`，先读取它。
4. 创建新页面前，先搜索是否已有相关页面。
5. 判断本次应新增页面、更新已有页面，还是两者都做。
6. 用结构化 Markdown 写入分析，并标注来源。
7. 为相关页面补充交叉引用。
8. 更新 `wiki/index.md`。
9. 在 `wiki/log.md` 追加简洁日志。
10. 汇报本次改动、使用的证据、沉淀的结论和仍需人工确认的问题。

## Ingest Prompt 模板

当维护者要求录入新材料时，使用以下模板：

```markdown
请按 eval_LLM_WiKi 维护规范 ingest 这个新材料：

Source:
- `raw/...`

请完成：
1. 阅读原始材料，提取关键事实。
2. 判断它属于哪类评测、哪类 badcase 或哪类模型行为。
3. 新增或更新 wiki 中相关页面。
4. 更新 wiki/index.md。
5. 更新 wiki/log.md。
6. 标出仍需人工确认的问题。
```

## Query Prompt 模板

当维护者要求基于 Wiki 回答问题时，使用以下模板：

```markdown
请基于 eval_LLM_WiKi 回答这个问题：

问题：
...

请先读取：
1. `wiki/index.md`
2. 与问题相关的 wiki 页面
3. 必要时再读取 raw/ 原始材料

回答要求：
- 引用相关页面或 raw 文件。
- 区分事实、推断和待确认内容。
- 如果产生了可复用结论，请建议应该写回哪个 wiki 页面。
```

## Lint Prompt 模板

当维护者要求做 Wiki 健康检查时，使用以下模板：

```markdown
请对 eval_LLM_WiKi 做一次 Wiki 健康检查。

请检查：
1. wiki/index.md 是否遗漏页面。
2. wiki/log.md 是否缺少最近操作。
3. 是否存在孤立页面。
4. 是否有同类 badcase 分散在多个页面但未互链。
5. 是否有旧结论被新材料推翻但未更新。
6. 是否有重要判断缺少 raw 来源。
7. 是否有页面命名、分类或格式不一致。

请输出：
- 问题列表
- 建议修复动作
- 优先级
```

## Badcase 页面模板

```markdown
# Badcase: 简短标题

## 摘要

一句话说明问题现象。

## 基本信息

- Source: `raw/...`
- Date: YYYY-MM-DD
- Model:
- Task:
- Category:
- Severity:

## 复现现象

描述模型实际输出和期望输出之间的差异。

## 原因分析

说明可能原因，例如 system prompt 约束不清、模型忽略上下文、评测样例歧义、工具调用失败或多轮状态污染。

## 影响范围

说明这个问题是否可能影响其他任务、语言、模型或评测集。

## 修复建议

列出可执行修复方向。

## 相关页面

- [[related-page]]

## 待确认

- 缺少哪些证据或人工判断。
```

## 日志模板

```markdown
## [YYYY-MM-DD] ingest | source-or-topic

- Source: `raw/...`
- Updated: `wiki/...`, `wiki/index.md`
- Summary: 本次新增或修正了什么。
- Needs confirmation: 仍需人工确认的问题。
```

## 汇报格式

每次任务结束时，汇报以下内容：

- 读取了哪些文件。
- 新建或更新了哪些文件。
- 提取了哪些关键事实。
- 哪些可复用结论已经写入 Wiki。
- 还有哪些开放问题或人工审核项。
