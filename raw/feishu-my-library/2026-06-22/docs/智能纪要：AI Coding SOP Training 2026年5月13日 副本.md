- **AI 编程培训概述**

  - **培训目的与对象**
  
    - 培训目的：钟武汨表示，本次 AI 编程的 SOP 培训面向公司全员，重点针对承担产研相关职责的同事，旨在让大家理解 AI 编程 SOP 背后的工作方式，使其更好地融入日常工作，提升效率、质量和协作体验。

  - **培训内容与目标**
  
    - 内容覆盖：培训将讲述公司 AI 编程的框架性 SOP，涵盖细致的实践要求，重点是让大家理解工作方式，而非纠结工具细节。
    - 目标设定：对于 AI 编程熟手，遵循 SOP 框架规则和基本规范；初学者按手册流程开展工作，建立稳定的 AI 使用习惯。后续还将组织经验分享和技术答疑，持续完善 SOP 手册。
    - 产研体系转型：AI 编程已成为产研工作的基础能力，公司产研体系将全面转向和拥抱 AI 编程，产研效能将以引入 AI 编程后的综合效率作为新的工作基线。

  - **验证与安全要求**
  
    - 明确 report 中应包含的基本验证线，如静态语法检查、动态语言的 lint 和 format 等，同时建立 smoke 或 e to e 的 test，确保影响用户路径或核心链路的请求得到行为验证。
    - 强调 API、key 等安全信息不能写入仓库，建议使用点 env 形式存储，降低泄露风险。
    - agent 需要一个隔离、安全可靠的 DEV 或 Sandbox 环境，以便进行真实验证，避免影响线上生产环境。
  - **工作方式转变**
  
    - 从传统的自己编码转变为定义任务并驱动 agent 完成任务，提升点在于问题的定义和验收的定义，如明确登录模块的安全加密方式等。
    - 可探索单线程执行和可控并行，如使用多 worktree 形式实现并行，但建议先掌握单线程单 init 的使用。
    - 尽可能保留与 init 相关的记录，如 plan、运行命令、baseline 等，便于后续验证和回滚。
- **AI 编程 report 体系**

  <grid>
  <column width-ratio="0.670000">
  - **report 层级结构**
  
    - 王志文将 init ready 的 report 分为 6 层，包括 map（仓库入口模块关系）、commands（服务 build 和 test 命令）、safe env（安全运行环境）、边界（红区模块 owner 和禁止事项）、证据（任务完成证据）和 memory（业务逻辑沉淀）。
    - 强调 report 中应沉淀有价值的信息，如业务特殊逻辑、回归测试用例、线上归因分析等，使 report 更加 inter ready。
  - **int MD 规范**
  
    - int MD 是规范，Codex、Claude code 等会将其作为上下文注入。它应包含 SPEC 读取要求、相关命令、红区路径等信息，且不能包含账号、TOKEN 等敏感信息。
    - 建议 int MD 按需读取，避免过长导致 engine 注意力分散。
  - **report playbook**
  
    - report playbook 是 report 中值得沉淀的内容，如本地环境或 Sandbox 环境搭建方法、e to e 和 smoke 测试运行方式、历史 bug 回归用例等。
    - 公司级别的 scale 可提供更多场景应用，如 bot lab 的 call debugging、日志平台封装等，让 agent 可根据上下文分析问题归因。
