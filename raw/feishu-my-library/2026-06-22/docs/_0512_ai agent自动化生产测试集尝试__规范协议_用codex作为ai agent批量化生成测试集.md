# [规范协议]用codex作为ai agent批量化生成测试集

**# 测试集生成约束（通用硬性标准）**



**## 1. 文件级约束**



1. 数据文件必须使用 `JSONL`（每行一个完整 JSON 对象）。

2. 文件编码必须为 \`UTF-8\` 且 **不带 BOM**。

1. 换行符统一使用 `\n`。
2. 文件中不允许空行；如必须存在，解析器需显式忽略空白行。
3. 单文件内每条样本 `id` 必须唯一。

**## 2. 顶层结构约束**



每条样本（每行 JSON）至少包含：



1. `id`：字符串，唯一标识。
2. `type`：字符串，推荐枚举（如 `single` / `tree`），不得缺失。
3. `dialog`：对象或数组（取决于 `type`），不得缺失。
4. `node_index`：当 `type=tree` 时必填。
5. `meta`：对象，至少包含基础描述信息（如语言、用途）。

**## 3. 树结构约束（type=tree）**



1. `node_index.root_node` 必须存在且能在 `dialog` 中找到。
2. `node_index.leaf_nodes` 必须是数组，且每个叶子节点都存在于 `dialog`。
3. 每个节点键（如 `t0-b0`）在单条样本内必须唯一。
4. 每个节点必须包含：

   - `turn_index`（整数，从零开始）
   - `role`（字符串）
   - `content`（字符串，可为空字符串但字段不能缺失）
   - `parent`（根节点为 `null`，其他节点为有效节点 id）
   - `children`（数组；叶子节点为 `[]`）
5. 任何 `children` 引用的节点，其 `parent` 必须回指当前节点。
6. 必须无孤儿节点：除根节点外，每个节点必须可从 `root_node` 可达。
7. 必须无环：父子关系必须构成 DAG，推荐为树/森林中的单树。
8. `turn_index` 应与路径顺序一致，沿父子路径单调递增。

**## 4. 对话轮次约束**



1. `role` 必须来自受控集合（如 `system`、`user`、`assistant`、`tool`）。
2. 相邻轮次不应出现不合理的同角色连续（若业务允许需在规范中声明）。
3. `content` 必须是可序列化纯文本；禁止嵌入未转义控制字符。
4. 若存在工具调用轮次，调用与响应必须成对、可追踪。

**## 5. 一致性与可评测约束**



1. 同一数据集内字段命名必须一致（不能混用别名）。
2. 可选字段若出现，类型必须稳定（不能同字段跨样本变更类型）。
3. 样本应可被“逐行解析 + 结构校验”无错误通过。
4. 不依赖隐式上下文：单条样本应自描述（至少包含必要上下文节点）。

**## 6. 质量门禁（发布前）**



发布前必须通过以下自动检查：



1. 编码检查：`UTF-8 no BOM`。
2. JSON 语法检查：逐行 `json parse`。
3. Schema 检查：必填字段、字段类型、枚举值。
4. 图一致性检查：

   - root/leaf 合法
   - parent/children 互相一致
   - 无孤儿、无环
5. 唯一性检查：`id` 唯一、节点键唯一。

**## 7. 版本与变更约束**



1. 数据集更新必须递增版本号（建议语义化版本）。
2. 任何结构性变更（字段增删、类型变化）必须记录 changelog。
3. 新旧版本应提供迁移说明或兼容策略。

**## 8. 不纳入本通用文档的内容**



以下内容属于“场景特定策略”，不应放入通用硬约束：



1. 某行业务话术模板。
2. 某语言/地区专属表达规则。
3. 某项目独有的流程分支或结束信号。
4. 某模型专属 prompt 细节。
